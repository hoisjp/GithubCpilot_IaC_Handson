# サンプルプロンプト: 生成 Bicep のレビュー

Step 4 で使う、Copilot に Bicep をレビューさせるプロンプトです。

---

あなたは Azure の FTE (シニアクラウドアーキテクト) です。
ワークスペース内の `bicep/` 配下にある Bicep テンプレートを、以下の観点でレビューしてください。

## 観点

### 1. Azure Well-Architected Framework (5 本柱)
- **信頼性** — SLA、冗長性、バックアップ、障害時の挙動
- **セキュリティ** — ID 管理、ネットワーク分離、データ保護、シークレット管理
- **コスト最適化** — SKU の妥当性、環境ごとのサイジング、不要リソース
- **運用の卓越性** — IaC 品質、観測可能性、デプロイ容易性、環境分離
- **パフォーマンス効率** — スケール戦略、ボトルネック、キャッシュ

### 2. Bicep ベストプラクティス
- `param` の `@description` / `@allowed` / `@minLength` / `@secure()` の使い分け
- 命名規則とグローバル一意性 (`uniqueString` の適切な使用)
- `tags` の伝播
- モジュール分割と責務、循環参照の回避
- `dependsOn` の最小化 (参照による自動解決)
- 出力に秘匿情報が含まれていないか
- API バージョンの新しさ
- RBAC ロール割り当ての `guid()` による冪等化

### 3. セキュリティベースライン
- Managed Identity によるパスワードレス化が徹底されているか
- 最小権限 (Contributor/Owner の乱用なし) か
- HTTPS / TLS 1.2 強制、FTP 無効化
- ストレージの public access 無効、共有キー無効
- Key Vault の RBAC モード、soft delete / purge protection
- SQL の Entra ID-only 認証
